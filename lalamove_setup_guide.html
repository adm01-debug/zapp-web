🚚 Lalamove Flows — Guia de Setup

Deploy e configuração dos 4 workflows no n8n · Promo Brindes

📦 Bundle incluído: 5 workflows prontos para importar — Bootstrap + F01 + F02 + F03 + F04

📋 Visão Geral dos Flows

FlowWebhookTriggerAção
🚚 F01 Cotação /webhook/lalamove-cotacao Automação B24: stage PREPARATION Geocode → Quotation API → grava quotationId + stopIds + preço no SPA
📦 F02 Pedido /webhook/lalamove-pedido Automação B24: stage APPROVAL Lê SPA → Order API → grava orderId → stage IN_PROCESS
📡 F03 Rastreamento /webhook/lalamove-eventos Webhook Lalamove Dashboard Valida HMAC → busca SPA pelo orderId → atualiza stage
🖼️ F04 POD /webhook/lalamove-pod Chamado pelo F03 após entrega GET Order detail → extrai fotos POD → stage WON no B24

⚙️ Passo 1 — Configurar Variáveis no n8n

Acesse: n8n → Settings → Variables e crie duas variáveis:

N8N_API_KEY
API key do próprio n8n — usada pelo Bootstrap para criar os flows
Onde criar: Settings → API → Create API Key
B24_TOKEN
Token do webhook de entrada do Bitrix24 — aparece na URL do webhook
Onde encontrar: Bitrix24 → Aplicativos → Webhooks de entrada → ID do webhook
Exemplo: URL https://promobrindes.bitrix24.com.br/rest/1/abc123xyz/crm.item.get → token = abc123xyz

📥 Passo 2 — Importar e Executar o Bootstrap

1No n8n, clique em Workflows → Import from File
2Selecione o arquivo lalamove_bundle_import.json
3O n8n importará 5 workflows. Abra o ⚙️ LLM-BOOTSTRAP
4Clique em ▶️ Execute Workflow — ele criará e ativará F01, F02, F03 e F04 automaticamente
5Verifique o output: todos os flows devem aparecer com status CREATED
⚠️ Alternativa manual: Se preferir, importe cada JSON separadamente (llm_f01.json ... llm_f04.json), ative cada um, e substitua $env.B24_TOKEN pelo token real nos Code nodes.

🔗 Passo 3 — Registrar Webhook no Lalamove

URL do Webhook

https://webhook.atomicabr.com.br/webhook/lalamove-eventos

Registrar em: Lalamove Dashboard → Configurações → Webhooks

Eventos a ativar

ORDER_ACCEPTED DRIVER_ARRIVED ORDER_PICKED_UP ORDER_DELIVERED ORDER_COMPLETED ORDER_CANCELLED DRIVER_REJECTED

🔄 Fluxo Completo

SPA criado
   │
   ├─ stage → PREPARATION
   │    └─ Automação B24 → POST /lalamove-cotacao
   │         └─ F01: Geocode → Quotation API → grava quotationId+stops+preço
   │
   ├─ stage → APPROVAL (aprovação humana)
   │    └─ Automação B24 → POST /lalamove-pedido
   │         └─ F02: Lê SPA → Order API → grava orderId → stage IN_PROCESS
   │
   ├─ [Motorista aceita] → Lalamove → POST /lalamove-eventos
   │    └─ F03: valida HMAC → find SPA → stage PREPARATION (em coleta)
   │
   ├─ [Motorista chega] → Lalamove → POST /lalamove-eventos
   │    └─ F03: stage IN_PROCESS
   │
   ├─ [Coleta realizada] → Lalamove → POST /lalamove-eventos
   │    └─ F03: stage IN_PROCESS
   │
   └─ [Entrega realizada] → Lalamove → POST /lalamove-eventos
        └─ F03: stage WON → dispara F04
             └─ F04: GET order detail → extrai fotos POD → stage WON + flags
✅ Tudo configurado? Crie um pedido real no Bitrix24, mova para PREPARATION e acompanhe os logs do n8n em tempo real.
